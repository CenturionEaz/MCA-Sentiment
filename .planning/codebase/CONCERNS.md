# Codebase Concerns

## AI Model Dependencies and Initialization
- **First Run Penalty**: The Python AI Service downloads ~2GB of HuggingFace models upon its first execution. This requires a stable internet connection and can cause timeouts or perceived hangs during initial setup.
- **Resource Intensity**: The models (RoBERTa, BART-large) require significant RAM/VRAM. On systems without a dedicated GPU (CUDA/MPS), CPU inference can be slow and memory-intensive.

## Performance and Scalability
- **Synchronous API Calls**: The Java backend calls the Python AI service synchronously during file uploads. For very large CSV files, this pipeline (especially abstractive summarization) could lead to HTTP timeouts or poor user experience.
- **Stateless AI Processing vs. State**: The Python API handles batches statelessly, but the Java backend must hold the entire dataset in memory to aggregate results, which could cause `OutOfMemoryError` on massive datasets.

## State Management and Concurrency
- **Static Assets**: Word clouds are generated by the Python service and saved statically. Care must be taken to ensure concurrent user sessions do not overwrite each other's visual assets, possibly requiring unique file naming based on session IDs or UUIDs.

## Dependency Management
- **Dual Ecosystems**: Maintenance requires managing dependencies across two distinct ecosystems: Maven (`pom.xml`) for Java and Pip (`requirements.txt`) for Python. Vulnerability scanning and updating must be performed on both independently.
- **Hardcoded Secrets**: Review `application.properties` and setup scripts to ensure no sensitive data (like Google OAuth Client Secrets) is accidentally committed to source control.
